Capabilities

Everything you need to orchestrate AI at scale

From multi-backend routing to session memory — nexusOrchestrator handles the infrastructure so you focus on building.

🔀

Multi-Backend Routing

Automatically discovers and routes tasks to LM Studio, Ollama, OpenAI, Anthropic, or any OpenAI-compatible endpoint. Per-task model selection with provider hints.

🧠

Session Memory

Every project path gets its own SQLite-backed conversation history. Multi-turn interactions maintain context so your LLM always knows what came before.

🔌

MCP Server

Built-in JSON-RPC 2.0 server on port 9998, compatible with Claude Desktop and any MCP client. Six tools ready to use out of the box.

🌐

Full REST API

Complete task management on port 9999 — submit, list, cancel, monitor. Real-time updates via Server-Sent Events for live progress tracking.

🛡️

Context Guard

Pre-flight token estimation prevents prompt overflow. Tasks exceeding context limits are flagged as TOO_LARGE immediately — before reaching the LLM.

⚡

Smart Discovery

Auto-detect running LLM backends with health checks and automatic failover. If your preferred provider goes down, the orchestrator finds an alternative.

🎯

Command Routing

Classify tasks as "plan" or "execute". The orchestrator enforces that execution tasks have a prior plan, preventing uncoordinated AI-generated changes.

🖥️

Desktop GUI

Native Wails desktop app with embedded HTTP API, real-time task monitoring, system tray integration, and a live provider health dashboard.

Quick Start

Up and running in 60 seconds

Build the daemon, start it, and submit your first task.

1

Clone & build

Requires Go 1.24+ and CGO enabled for SQLite.

git clone https://github.com/el-j/nexusOrchestrator
cd nexusOrchestrator
CGO_ENABLED=1 go build -o nexus-daemon ./cmd/nexus-daemon/...
2

Start the daemon

The daemon starts the HTTP API, MCP server, and auto-discovers local LLM backends.

./nexus-daemon
# HTTP API  →  http://localhost:9999
# MCP       →  http://localhost:9998/mcp
# Dashboard →  http://localhost:9999/ui
3

Submit your first task

Send a POST request with your project path and instruction.

curl -X POST http://localhost:9999/api/tasks \
  -H "Content-Type: application/json" \
  -d '{
    "projectPath": "/path/to/your/project",
    "targetFile":  "output.go",
    "instruction": "Write a function that sorts strings"
  }'
4

Check task status

Poll the task endpoint or use Server-Sent Events for real-time updates.

curl http://localhost:9999/api/tasks/{id}
Response JSON 
{
  "id": "a1b2c3d4-e5f6-7890-abcd-ef1234567890",
  "projectPath": "/path/to/your/project",
  "targetFile": "output.go",
  "instruction": "Write a function that sorts strings",
  "status": "QUEUED",
  "command": "auto",
  "createdAt": "2025-01-01T00:00:00Z",
  "updatedAt": "2025-01-01T00:00:00Z"
}
Status after completion JSON 
{
  "id": "a1b2c3d4-...",
  "status": "COMPLETED",
  "logs": "func sortStrings(s []string) []string {\n  sort.Strings(s)\n  return s\n}",
  "updatedAt": "2025-01-01T00:00:04Z"
}

Architecture

Hexagonal architecture, strictly enforced

Core business logic never imports adapter code. All external concerns are abstracted behind interfaces.

Inbound Adapters  |  HTTP API (chi)  ·  MCP (JSON-RPC)  ·  CLI  ·  Wails GUI
↓                                                           ↓
Core Services  |  OrchestratorService  ·  DiscoveryService
↓                                                           ↓
Port Interfaces  |  Orchestrator  ·  LLMClient  ·  TaskRepo  ·  FileWriter  ·  Session
↓                                                           ↓
Outbound Adapters  |  LM Studio  ·  Ollama  ·  OpenAI  ·  Anthropic  ·  SQLite  ·  FS
