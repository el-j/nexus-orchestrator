Getting Started

Install nexusOrchestrator, configure your LLM backends, and submit your first task in minutes.

Prerequisites

Before installing nexusOrchestrator, make sure you have the following:

🐹

Go 1.24+

Required to build from source. Download from go.dev

go version
🔧

CGO Enabled

Required for the SQLite session store. Set CGO_ENABLED=1 before building.

export CGO_ENABLED=1
🦙

An LLM Backend

At least one of: LM Studio, Ollama, or API keys for OpenAI / Anthropic.

🐙

Git

To clone the repository. Or download a pre-built binary from the downloads page.

Installation

1

Clone the repository

Clone nexusOrchestrator from GitHub:

git clone https://github.com/el-j/nexusOrchestrator.git
cd nexusOrchestrator
2

Install dependencies

Go modules are used — dependencies are fetched automatically:

go mod download
3

Build the daemon

CGO must be enabled for SQLite support:

CGO_ENABLED=1 go build \
  -o nexus-daemon \
  ./cmd/nexus-daemon/...
💡

On macOS with Homebrew, ensure Xcode Command Line Tools are installed: xcode-select --install

4

Set environment variables (optional)

Configure API keys for cloud providers:

export OPENAI_API_KEY=sk-...
export ANTHROPIC_API_KEY=sk-ant-...
export GITHUB_TOKEN=ghp_...
5

Start the daemon

Launch nexus-daemon — it will auto-discover local LLM backends:

./nexus-daemon

You should see:

 ✓ HTTP API   → http://localhost:9999
 ✓ MCP Server → http://localhost:9998/mcp
 ✓ Dashboard  → http://localhost:9999/ui

⚡ Quick Verify

Once the daemon is running, verify it's healthy:

curl http://localhost:9999/health

Expected response:

{"status": "ok"}

🔍 Check Providers

See which LLM backends were discovered:

curl http://localhost:9999/api/providers

Environment Variables

nexusOrchestrator reads the following environment variables at startup:

Variable Default Description
NEXUS_HTTP_PORT 9999 Port for the HTTP REST API and dashboard
NEXUS_MCP_PORT 9998 Port for the MCP JSON-RPC server
NEXUS_DB_PATH ~/.nexus/sessions.db Path to the SQLite session database
NEXUS_LOG_LEVEL info Log verbosity: debug, info, warn, error
OPENAI_API_KEY — OpenAI API key — enables the OpenAI provider
ANTHROPIC_API_KEY — Anthropic API key — enables the Anthropic provider
GITHUB_TOKEN — GitHub token — enables the GitHub Copilot provider
NEXUS_LM_STUDIO_URL http://127.0.0.1:1234 Override LM Studio endpoint (if running on a different host/port)
NEXUS_OLLAMA_URL http://127.0.0.1:11434 Override Ollama endpoint
NEXUS_CONTEXT_LIMIT 128000 Default token limit for context-window guard

Your First Task

Submit a code-generation task and poll for the result:

1. Submit the task

curl -X POST http://localhost:9999/api/tasks \
  -H "Content-Type: application/json" \
  -d '{
    "projectPath": "/Users/you/myproject",
    "targetFile":  "utils/sort.go",
    "instruction": "Write a generic sort function for slices",
    "command":     "execute"
  }'

You'll receive a task ID in the response:

{
  "id": "abc-123-def-456",
  "status": "QUEUED"
}

2. Poll for completion

curl http://localhost:9999/api/tasks/abc-123-def-456

When complete, the response includes the generated code:

{
  "id": "abc-123-def-456",
  "status": "COMPLETED",
  "logs": "package utils\n\nimport \"sort\"\n\nfunc SortSlice[T any](s []T, less func(a, b T) bool) {\n\tsort.Slice(s, func(i, j int) bool {\n\t\treturn less(s[i], s[j])\n\t})\n}\n"
}
ℹ️

Real-time updates: Instead of polling, subscribe to Server-Sent Events at GET /api/tasks/{id}/stream for live progress updates.

Web Dashboard

The daemon ships with an embedded web dashboard at http://localhost:9999/ui. Open it in your browser to:

📋

Task Queue

View all QUEUED and PROCESSING tasks with live status updates, instructions, and timing.

🔌

Provider Status

See which LLM backends are active, their current models, and health check results.

📜

Task History

Browse completed tasks, view generated outputs, and inspect error logs.
