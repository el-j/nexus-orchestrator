MCP Integration

Connect Claude Desktop or any MCP client to nexusOrchestrator using the built-in JSON-RPC 2.0 server on port 9998.

What is MCP?

The Model Context Protocol (MCP) is an open standard for connecting AI assistants to external tools and data sources. nexusOrchestrator implements an MCP server using JSON-RPC 2.0, making it compatible with Claude Desktop and any MCP-aware client.

🔌

JSON-RPC 2.0

The MCP server uses the industry-standard JSON-RPC 2.0 protocol over HTTP, making it easy to integrate with any language or framework.

🤖

Claude Desktop

Add nexusOrchestrator as a tool provider in Claude Desktop with a single config change. All six tools become available in Claude's tool palette.

🛠️

6 Tools Ready

Submit tasks, check status, list the queue, cancel tasks, inspect providers, and check health — all via MCP tool calls.

Claude Desktop Setup

Add nexusOrchestrator to your Claude Desktop configuration file. The location depends on your operating system:

⚠️

Start the daemon first. Make sure nexus-daemon is running before starting Claude Desktop. The MCP server must be listening on port 9998.

~/Library/Application Support/Claude/claude_desktop_config.json 
{
  "mcpServers": {
    "nexusOrchestrator": {
      "url": "http://localhost:9998/mcp"
    }
  }
}
%APPDATA%\Claude\claude_desktop_config.json 
{
  "mcpServers": {
    "nexusOrchestrator": {
      "url": "http://localhost:9998/mcp"
    }
  }
}
~/.config/Claude/claude_desktop_config.json 
{

      "url": "http://localhost:9998/mcp"
    }
  }
}
1

Start the daemon

Make sure nexus-daemon is running before configuring Claude.

./nexus-daemon
# MCP Server → http://localhost:9998/mcp
2

Edit the config file

Open your Claude Desktop config at the path shown above and add the mcpServers block.

3

Restart Claude Desktop

Quit and reopen Claude Desktop. The nexusOrchestrator tools should appear in the tool palette (🔧 icon).

4

Test the connection

Ask Claude to check the orchestrator health or list providers:

"Use the nexusOrchestrator health tool to check if the daemon is running"

Available Tools

Tool Description Arguments
submit_task Submit a code-generation task to the orchestrator queue projectPath*, targetFile*, instruction*, command, modelId, providerHint
get_task Retrieve the status and output of a task by its ID id*
get_queue List all pending (QUEUED/PROCESSING) tasks —
cancel_task Cancel a queued task before it is processed id*
get_providers List all registered LLM providers with health status —
health Check if the orchestrator daemon is running —

* Required argument

Usage Examples

Raw JSON-RPC 2.0 request/response examples for each tool:

submit_task

Request JSON-RPC 
{
  "jsonrpc": "2.0",
  "id": 1,
  "method": "tools/call",
  "params": {
    "name": "submit_task",
    "arguments": {
      "projectPath": "/Users/me/myapp",
      "targetFile":  "auth/middleware.go",
      "instruction": "Add JWT authentication middleware",
      "command":     "execute",
      "providerHint":"LM Studio"
    }
  }
}
Response JSON-RPC 
{
  "jsonrpc": "2.0",
  "id": 1,
  "result": {
    "content": [{
      "type": "text",
      "text": "{\"id\":\"task-abc-123\",\"status\":\"QUEUED\",\"createdAt\":\"2025-01-01T00:00:00Z\"}"
    }]
  }
}

Troubleshooting

Problem Cause Solution
Tools don't appear in Claude Daemon not running when Claude started Start nexus-daemon first, then restart Claude Desktop
Connection refused on :9998 Daemon crashed or wrong port Check daemon logs; verify NEXUS_MCP_PORT matches config URL
health returns error SQLite not initialized or CGO missing Rebuild with CGO_ENABLED=1; check write permissions for DB path
Tasks stuck in QUEUED No active LLM provider Start LM Studio / Ollama or set API keys; call get_providers to check status
TOO_LARGE status Prompt exceeds context window Reduce contextFiles, shorten instruction, or increase NEXUS_CONTEXT_LIMIT
JSON parse error in response Malformed request Ensure jsonrpc: "2.0" and required id, method, params fields are present
Method not found Wrong method name Use exactly tools/call for tool invocation; check tool name spelling
💡

Debug mode: Start the daemon with NEXUS_LOG_LEVEL=debug ./nexus-daemon to see detailed request/response logging for all MCP calls.

ℹ️

Verify MCP is listening: curl http://localhost:9998/mcp -X POST -H "Content-Type: application/json" -d '{"jsonrpc":"2.0","id":1,"method":"tools/list","params":{}}'

Using Other MCP Clients

nexusOrchestrator's MCP server is standards-compliant and works with any JSON-RPC 2.0 client. You can also call it directly from scripts:

# Test MCP health from the command line
curl -s -X POST http://localhost:9998/mcp \
  -H "Content-Type: application/json" \
  -d '{
    "jsonrpc": "2.0",
    "id": 1,
    "method": "tools/call",
    "params": {
      "name": "health",
      "arguments": {}
    }
  }' | jq '.result.content[0].text | fromjson'
# Submit a task via MCP from the command line
curl -s -X POST http://localhost:9998/mcp \
  -H "Content-Type: application/json" \
  -d '{
    "jsonrpc": "2.0",
    "id": 2,
    "method": "tools/call",
    "params": {
      "name": "submit_task",
      "arguments": {
        "projectPath": "'$(pwd)'",
        "targetFile": "generated.go",
        "instruction": "Write a Hello World HTTP handler"
      }
    }
  }' | jq '.result.content[0].text | fromjson'
